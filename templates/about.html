An information system for collecting user-defined information from the Internet

Technical specification for an information system for collecting user-defined information from the Internet:

1. The purpose of the project:
- Collect and structure information from specified Internet resources specified by a specific user.

2. Functional requirements:
- The user should be able to specify the Internet resources from which information should be collected.
- The user should be able to specify criteria and filters for collecting information.
- The system should be able to automatically collect and update information from the specified resources.
- The system should be able to structure and display the collected information for the convenience of the user.

3. Non-functional requirements:
- The program must be written in the Python programming language.
- To collect information from the Internet, use the Requests and BeautifulSoup libraries.
- The data must be stored in a MySQL database.
- The user interface should be implemented as a web application using the Flask framework.
- The system must be protected from unauthorized access.

4. Testing Requirements:
- To test the collection of information from various Internet resources.
- Check the correctness of the display and structuring of information.
- Check the performance of the system with a large amount of data.

5. Work plan:
- Development of the system architecture and database.
- Writing scripts to collect information from Internet resources.
- Development of a web-based user interface.
- Testing and debugging of the system.
- System implementation and user training.

6. Risk assessment:
- Unavailability of Internet resources for collecting information.
- Copyright infringement in the collection of information.

7. Update and Support plan:
- Carrying out regular updates to maintain the health of the system.
- Support and training of users on the use of the system.
